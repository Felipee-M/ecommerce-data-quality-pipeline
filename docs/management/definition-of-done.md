# Definition of Done

A task is considered done only when it meets the following criteria:

## General Criteria

- The deliverable is completed according to the sprint objective.
- The project structure remains organized.
- Files are versioned in Git.
- Documentation is updated when necessary.
- The change adds clear value to the project.

## Code Criteria

- Code runs without errors.
- Naming is clear and consistent.
- Sensitive information is not exposed.
- Environment variables are documented in `.env.example`.
- The code follows the project folder structure.

## Data Criteria

- Data source is documented.
- Raw data files are not committed to GitHub.
- Data transformations are traceable.
- Quality rules are documented when applicable.

## Testing Criteria

- Applicable tests are created or updated.
- Tests are executed before closing the task.
- Test evidence is documented when relevant.

## Quality Criteria

- Nonconformities are documented when found.
- Corrective actions are proposed for relevant issues.
- Improvements are registered in the continuous improvement log.
